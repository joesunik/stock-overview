## 4️⃣ 데이터 저장 흐름

### 시나리오: 계좌 등록 (`addAccount()`)

```
Step 1: addAccount("51824747")
  ↓
Step 2: accountRepository.save(account) → ID 저장 (예: ID=1)
  ↓
Step 3: fetchAndSaveFromKiwoom(accountId=1, acctNo="51824747", startDate=2015-01-01, endDate=2026-02-05)
  ↓
Step 4: 월별 루프 (약 132개월)

  For each YearMonth (2015-01 ~ 2026-02):
    ├─ 월의 마지막 영업일 계산 (예: 2015-01-30 금요일)
    ├─ ka01690 호출: 국내주식 데이터
    ├─ kt00015 호출: 해외주식 데이터 (단일 날짜)
    ├─ kt00016 호출: 선물/수익률 데이터
    ├─ mergeAllData() → DailyBalance 엔티티 생성
    └─ dailyBalanceRepository.save()

  저장되는 필드:
  ✅ estimatedAsset
  ✅ depositBalance
  ✅ totalEvltAmt (국내주식)
  ✅ profitRate
  ✅ foreignStockAmt (해외주식) ✨
  ✅ futuresAmt (선물) ✨
  ✅ totalAssetAmt (합계) ✨

Step 5: 저장 완료 (월별 데이터 약 132건)
```

### 시나리오: 해외/선물 데이터만 갱신 (`refreshForeignAndFuturesData()`)

```
Step 1: refreshForeignAndFuturesData("51824747", 2018-01-01, 2026-02-05)
  ↓
Step 2: 1년 단위 배치 처리

  For each year (2018 ~ 2026):
    ├─ kt00015 호출: 1년 범위 (예: 2018-01-01 ~ 2018-12-31)
    ├─ 거래내역 날짜별 그룹핑
    │
    └─ 월별 루프 (해당 연도)
        └─ kt00016 호출: 월별 마지막 영업일 (예: 2018-01-31)
           └─ 기존 DailyBalance 엔티티 찾기
              └─ foreignStockAmt, futuresAmt, totalAssetAmt 업데이트
              └─ dailyBalanceRepository.save()

Step 3: API 호출 최적화
  - 기존: 월별 호출 = kt00015 × 132 + kt00016 × 132 = 264회
  - 신규: 1년 단위 = kt00015 × 9 + kt00016 × 132 = 141회
  - 감소율: 46% ✨
```

## 5️⃣ API 호출 체계

### 프론트 → 백 API 호출 경로

```
프론트엔드 (accountApi.js)
  ↓
  normalizeDate() → YYYY-MM-DD
  ↓
  buildParams() → 쿼리 스트링
  ↓
  fetch(url) → GET /api/account/...?startDate=...&endDate=...&acctNo=...
  ↓
백엔드 (AccountController.java)
  ↓
  @DateTimeFormat(iso = DateTimeFormat.ISO.DATE) → LocalDate 파싱
  ↓
  AccountService 메서드 호출
  ↓
  DB 조회 (DailyBalance)
  ↓
  DTO 변환 후 응답
  ↓
프론트엔드 (parseResponse)
  ↓
  결과 렌더링
```

### 백 → Kiwoom API 호출 경로 (계좌 추가/갱신 시에만)

```
AccountService
  ↓
  fetchAndSaveFromKiwoom() 또는 refreshForeignAndFuturesData()
  ↓
  KiwoomDailyBalanceClient.fetchDailyBalance()
    → POST /api/dostk/acnt (ka01690)
    → Request: Ka01690Request (acctNo, qryDt)
    → Response: Ka01690Response
  ↓
  KiwoomForeignStockClient.fetchForeignStockHistory()
    → POST /api/trst/ovrl/trde (kt00015)
    → Request: Kt00015Request (acctNo, strtDt, endDt, gdsTp="3", ...)
    → Response: Kt00015Response
  ↓
  KiwoomAccountProfitClient.fetchAccountProfit()
    → POST /api/rlsn/daily/acnt (kt00016)
    → Request: Kt00016Request (acctNo, frDt, toDt)
    → Response: Kt00016Response
  ↓
  mergeAllData() → DailyBalance 엔티티
  ↓
  dailyBalanceRepository.saveAll()
```

## 6️⃣ 핵심 검증 포인트

### ✅ 날짜 포맷팅 검증

| 계층 | 포맷 | 예시 |
|------|------|------|
| **프론트엔드** | YYYY-MM-DD | 2018-01-01 |
| **HTTP 쿼리** | YYYY-MM-DD | ?startDate=2018-01-01 |
| **Spring 파싱** | LocalDate | LocalDate.of(2018, 1, 1) |
| **서비스** | yyyyMMdd | 20180101 |
| **Kiwoom API** | yyyyMMdd | strt_dt=20180101 |

### ✅ 필드 매핑 검증

#### Kt00015Request (해외주식)
```
@JsonProperty("acct_no") → String acctNo ✅
@JsonProperty("strt_dt") → String strtDt ✅ (이전 버그: stDt ❌)
@JsonProperty("end_dt") → String endDt ✅ (이전 버그: edDt ❌)
@JsonProperty("gds_tp") → String gdsTp ✅
@JsonProperty("crnc_cd") → String crncCd ✅
@JsonProperty("frgn_stex_code") → String frgnStexCode ✅
```

#### Kt00016Request (선물)
```
@JsonProperty("fr_dt") → String frDt ✅
@JsonProperty("to_dt") → String toDt ✅
```

### ✅ 데이터 병합 검증

```
mergeAllData():
  - domestic.totalEvltAmt → DailyBalance.totalEvltAmt ✅
  - foreign.trstOvrlTrdePrpsArray[].exctAmt → calculateForeignStockAmount() ✅
  - profit.futrReplSella → DailyBalance.futuresAmt ✅
  - totalEvltAmt + foreignStockAmt + futuresAmt → DailyBalance.totalAssetAmt ✅
```

## 8️⃣ 특수 기능: "해외/선물 갱신" 버튼

### 프론트엔드 `App.jsx`

```javascript
const handleRefreshForeignAndFutures = async () => {
  setRefreshForeignLoading(true);
  try {
    // 1. 백엔드 갱신 API 호출
    await apiRefreshForeignAndFutures(acctNo, searchParams.startDate, searchParams.endDate);

    // 2. 갱신 후 데이터 재조회
    const [summaryRes, dwRes, txRes] = await Promise.all([
      fetchMonthlySummary(...),
      fetchDepositWithdrawalSummary(...),
      fetchTransactions(...)
    ]);

    // 3. UI 업데이트
    setMonthlySummary(summaryRes);
    setDepositWithdrawalSummary(dwRes);
    setTransactions(txRes);
  } finally {
    setRefreshForeignLoading(false);
  }
};
```

### 백엔드 `AccountController.java`

```java
@PostMapping("/refresh-foreign-futures")
public ResponseEntity<Map<String, String>> refreshForeignAndFuturesData(
    @RequestParam String acctNo,

    @RequestParam @DateTimeFormat(iso = DateTimeFormat.ISO.DATE) LocalDate endDate
) {
    accountService.refreshForeignAndFuturesData(acctNo, startDate, endDate);

}
```
